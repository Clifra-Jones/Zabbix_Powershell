Get-ZabbixAuthCode

Authorize use of the API.

Login and receive an authorization token.
User must have a Zabbix logon.
If security settings for the user have changed, a new authorization token will be required.
If username and password are omitted they will be prompted for.

Get-ZabbixAuthCode Aliases

  • Connect-ZabbixUser

Syntax 

Get-ZabbixAuthCode [[-Uri] <String>] [[-Username] <String>] [[-Password] <SecureString>] [[-ProfileName] <String>] [-StoreToProfile] [<CommonParameters>]

Parameters

Name Alias Type Description Required? Pipeline Input Default Value
-Uri String The Zabbix API Uri for you primary Zabbix Server. false false
-Username String Username to log in with. false false
-Password SecureString Password to log in with. This MUST be a secure string. false false
-ProfileName String The name to store the profile in. Must be used with the StoreToProfile Parameter.
If omitted the profile is stored as the default profile.		 false false
-StoreToProfile SwitchParameter Stores the authorization token in a configuration file in the directory .zabbix in the user's profile. false false False

Outputs

The output type is the type of the objects that the cmdlet emits.

  • The authorization token as a string.

Note

This command As the alias Connect-ZabbixUse. Eventually the Get-ZabbixAuthcode command will be depreciated
in favor of Connect-ZabbixUser.

The authcode will remain active until you log out with the Disconnect-ZabbixUser function (alias: Remove-ZabbixAuthCode)
unless the user has the autologoff property set.

If your authcode is saved in a profile and still active DO NOT execute this command again. Use the stored profile. This will
prevent multiple user sessions from being created.

If you execute this command without the StoreToProfile and ProfileName parameters the auth token will only be retained for the current session.
You may want to use this if you are using a SuperAdmin account to perform some functions and don't want
that authcode stored in you profile file.
Failing to disconnect in this situation will leave the session active if the user account does not have an autologoff property set.
Closing the powershell window WILL NOT log the user off. Neither will executing a Remove-Module command.

Invoke-ZabbixAPI

Make a custom call to the zabbix API.

This method allows you to make a customized call to the API.
There are many parameters for the API calls. Not all of them are in the module functons. This is for simplicity sake.

Syntax 

Invoke-ZabbixAPI [-Method] <String> [-params] <PSObject> [[-ProfileName] <String>] [[-AuthCode] <String>] [[-Uri] <String>] [<CommonParameters>]

Parameters

Name Alias Type Description Required? Pipeline Input Default Value
-Method String This is the method to call. i.e. 'host.get' true false
-params PSObject This is an object containing the parameters to use. true false
-ProfileName String false false
-AuthCode String false false
-Uri String false false

Outputs

The output type is the type of the objects that the cmdlet emits.

  • Return the result of the call. This is the complete response. It is up to you to display or use the data.

Examples

EXAMPLE 1 
This example calls the hostgroup.get method and returns groups with a hosts property containing hostid and name.
$params = @{
    selectHosts = @(
        "hostid",
        "name"
    )
}
$response  = Invoke-ZabbixAPI -Method "hostgroup.get" -params $params
EXAMPLE 2 
This example shows how to search hosts by host tags. Only output hostid and host name.
$params = @{
    output = @(
